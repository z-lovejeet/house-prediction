2. The Machine Learning Pipeline

The model was trained on the Bengaluru House Price Dataset (9,200+ samples). The pipeline guarantees clean data and optimal model performance through 7 distinct phases.

Phase 1-3: Data Cleaning

  • Drop NaNs: Removed incomplete rows.
  • Standardize Sqft: Converted range values (e.g., "2100 - 2850") to exact averages.
  • Outlier Removal: Removed houses where the price-per-sqft was beyond 1 Standard Deviation from the mean.

Phase 4-5: Feature Engineering

  • Standardization: Added price_per_sqft for math logic.
  • Dimensionality Reduction: Any location with fewer than 10 houses was grouped into a single "Other" category to prevent the One-Hot array from exploding in size.

Phase 6-7: Optimization & Training

  • Algorithms: Trained Linear Regression, Ridge, Lasso, and ElasticNet.
  • GridSearchCV: Used 5-fold cross-validation to test hundreds of hyperparameters.
  • Result: ElasticNet was chosen as the champion model with an Alpha of 0.001 and l1_ratio of 0.8.

3. Behind the Scenes: The Mathematics

Linear Regression Pipeline

The baseline model fits a mathematical line describing the relationship between inputs and price.

Price = (w₁ × Area) + (w₂ × BHK) + (w₃ × Bath) + Σ(wₖ × Location) + Intercept

Why we used L2 Regularization (Ridge/ElasticNet)

In standard linear regression, with 262 features (mostly locations), the model suffers from Overfitting. It might assign an absurd weight like +₹50 Crores to a rare neighborhood just because of one anomalous data point. This results in Unstable Coefficients.

L2 Regularization solves this by adding a "penalty" to the loss function based on the size of the coefficients (weights).

Loss = Standard MSE Error + α · Σ(Weights²)

What this does: It physically pulls the weights down towards zero. The algorithm is forced to spread the value out logically rather than putting massive weights on single features. This perfectly satisfies the teacher's requirement: Minimize regularized loss (L2) to achieve stable coefficients.

Key Terminology Dictionary

Term Definition in our Project
MSE (Mean Squared Error) The average squared difference between the predicted prices and actual prices. We mathematically minimize this to train the model.
R² (R-Squared) Score The accuracy grade. An R² of 0.71 means our model explains 71% of the variance in house prices. (A very high score for messy real-estate data).
Alpha (\(\alpha\)) The severity of the regularization penalty. High alpha = strict penalty (underfitting). Low alpha = weak penalty (overfitting). We automatically found the perfect balance (\(\alpha=0.001\)) using GridSearchCV.
Learning Rate Trick question! Scikit-learn's standard estimators use closed-form mathematical solvers (like SVD or Cholesky) or Coordinate Descent, not traditional Gradient Descent, meaning there is no explicit "learning rate" like in deep learning. The hyperparameter we tune is Alpha.

4. Presentation Q&A Guide (Viva Prep)

If the examiner asks you any questions about the project, use these exact, pre-prepared answers to demonstrate your deep understanding.

Q: Did you achieve the objective of "Minimizing regularized loss (L2) for stable coefficients"?
Answer: Yes, perfectly. We implemented Ridge and ElasticNet which explicitly add an L2 penalty to the loss function. Before L2, our Linear Regression baseline was mathematically unstable—rare locations were getting massive negative or positive coefficients. By applying L2 regularization, we forced the model to shrink and stabilize those coefficients, preventing overfitting and giving us highly realistic location weights.
Q: What is the final accuracy of your model?
Answer: Our final ElasticNet model achieved an R² score of 0.7158 (71.58%). Considering a dataset spanning 9,200 varying real-world homes, achieving >70% accuracy on unseen test data while maintaining strictly regularized coefficients is an excellent result.
Q: Why ElasticNet instead of just Ridge (L2)?
Answer: Our dataset had 260+ features after One-Hot Encoding the locations. Ridge (L2) shrinks coefficients but keeps all 260 features. ElasticNet combines L2 (for stability) with L1 (Lasso, which drops irrelevant features entirely). Our ElasticNet dropped 11 useless location features completely, making the model faster, smaller, and more stable than pure Ridge.
Q: How do you know how the model arrived at a specific price limit in the UI?
Answer: We built an Explainability Engine into the backend API. When predicting, the API returns the exact Feature Contributions. It mathematically extracts the exact monetary value (in Lakhs) that the Area, BHK, and specific Location added or subtracted from the Base Intercept. We display this dynamically to the user in a Waterfall chart.
